# 대한산업AI v3 UI 점검 체크리스트

검수일: 2026-05-24  
검수 대상: `/v3`, `/v3/products`, `/v3/company`, `/v3/mvp`, `/v3/contact`  
검수 뷰포트: Desktop `1440x900`, `1440x1100` / Mobile `390x900`, `390x1100`

> 참고: 화면 좌하단의 검은색 `N` 배지는 Next.js dev indicator라 실제 배포 UI 문제는 아니다.

## 총평

현재 v3는 라우트 분리와 기본 반응형은 안정적이다. 큰 깨짐이나 글자 겹침은 보이지 않는다. 다만 모든 페이지가 같은 히어로 구조와 같은 카드 스타일을 반복해서, “제대로 된 법인/고급 솔루션 업체”라는 인상은 아직 약하다.  
핵심 개선 방향은 `페이지별 시각 언어 차별화`, `모바일 히어로 압축`, `카드/섹션 밀도 조정`, `실제 업무 도식 강화`다.

## P0. 먼저 고칠 것

- [ ] 모든 하위 페이지의 히어로 비주얼이 거의 동일하다.
  - 현재: Home, Products, Company, MVP, Contact 모두 같은 `DAEHAN AI SYSTEM + 87건 + 막대 그래프` 패널을 반복한다.
  - 문제: 페이지를 이동해도 새 정보가 열리는 느낌이 약하고, 템플릿처럼 보인다.
  - 개선: 페이지별 대표 도식을 다르게 만든다.
    - Home: `수작업 입력 → 대한산업AI → ERP/문서/대시보드 산출` 큰 흐름도
    - Products: 모듈형 아키텍처 맵
    - Company: 방문 진단/분석/구현 프로세스
    - MVP: 후보 선정 → 데모 → 검증 퍼널
    - Contact: 문의폼/상담 흐름 중심

- [ ] 모바일 히어로가 너무 길다.
  - 현재: 모바일에서 히어로 카피, 버튼, 큰 비주얼 패널이 세로로 쌓여 첫 핵심 섹션 진입이 늦다.
  - 문제: 사용자가 제품/회사/MVP 내용을 보기 전에 같은 비주얼을 오래 지나가야 한다.
  - 개선: 모바일에서는 히어로 비주얼을 `compact` 버전으로 줄인다.
    - 막대 그래프 하단 영역 숨김
    - 3단 카드 높이 축소
    - 히어로 전체 높이 1스크롤 이내로 조정

- [ ] 메인 페이지의 핵심 도식이 첫 화면 아래에 있다.
  - 현재: 첫 화면 오른쪽 비주얼은 예쁜 대시보드 모형이지만, 실제 “무엇을 자동화하는지”는 아래 섹션에서 나온다.
  - 문제: 사용자가 처음 보는 장면에서 대한산업AI의 업무 구조가 바로 이해되지 않는다.
  - 개선: 메인 히어로 오른쪽을 실제 자동화 도식으로 교체하거나, 현재 비주얼 안에 `ERP / 메일 / 웹하드 / 엑셀 → AI 자동화 → 보고서 / 대장 / 대시보드`를 명확히 넣는다.

- [ ] 하위 페이지 히어로 제목 줄 수가 과하게 많다.
  - 현재: Company 히어로는 데스크톱에서도 5줄 이상으로 떨어지고, 모바일에서도 거의 모든 페이지가 큰 제목으로 화면을 많이 차지한다.
  - 문제: 고급스러운 느낌보다 다소 무거운 느낌이 난다.
  - 개선: 서브페이지 H1은 더 짧게 만들고 폰트 크기를 메인보다 낮춘다.
    - 예: `기업 업무 자동화를 구현하는 기술 파트너`
    - 예: `내부 시스템에 붙는 AI 자동화 제품군`

## P1. 시각 밸런스 개선

- [ ] 섹션 여백이 전반적으로 커서 정보 밀도가 낮다.
  - 현재: 데스크톱에서 섹션 간 여백이 넓고 카드 내부 공백도 많다.
  - 문제: 깔끔하긴 하지만 랜딩 초안처럼 비어 보이는 구간이 생긴다.
  - 개선: 기본 섹션 패딩을 줄이고, 카드 내부 `min-height`를 콘텐츠 길이에 맞게 재조정한다.

- [ ] 카드 스타일이 너무 반복된다.
  - 현재: 흰색 카드 + 작은 파란 아이콘 + 짧은 텍스트 구성이 거의 모든 페이지에서 반복된다.
  - 문제: 제품/사례/회사/프로세스의 정보 위계가 비슷해 보여 페이지별 성격이 약하다.
  - 개선: 카드 종류를 3개 정도로 나눈다.
    - 모듈 카드: 기능 설명 + 체크 리스트
    - 사례 카드: 산업 태그 + 업무 전/후 도식
    - 프로세스 카드: 번호 중심의 타임라인

- [ ] Capability 카드가 데스크톱에서 지나치게 비어 보인다.
  - 현재: 회사소개와 메인 하단의 역량 카드가 넓고 높지만 텍스트가 짧다.
  - 문제: 콘텐츠가 빈약해 보일 수 있다.
  - 개선: 카드 높이를 줄이거나, 각 역량에 구체 예시를 1줄 추가한다.
    - 예: `웹하드 폴더 감시`, `보고서 양식 자동 채움`, `월별 누락 건수 집계`

- [ ] 신뢰 로고 영역이 텍스트만 있어 임시 느낌이 강하다.
  - 현재: `Samsung Electronics`, `SK hynix` 등이 단순 텍스트로 나열된다.
  - 문제: 실제 도입사 로고처럼 보이지 않고 임의 텍스트 느낌이 난다.
  - 개선: 아직 실제 로고가 없다면 `검토 기업 예시`, `산업별 적용 가능 영역`처럼 표현을 바꾸거나 흐릿한 로고 placeholder 스타일로 만든다.

- [ ] CTA 밴드가 페이지마다 비슷하다.
  - 현재: 하단 CTA는 대부분 짙은 배경 + 문의 버튼 구조다.
  - 문제: 페이지별 행동 유도가 구분되지 않는다.
  - 개선:
    - Products: `우리 시스템 연결 가능성 확인`
    - Company: `방문 진단 요청`
    - MVP: `반복 업무 1개 보내기`
    - Contact: 문의폼 자체가 CTA이므로 하단 CTA는 약하게 처리

## P1. 문구와 정보 구조

- [ ] 메인에서 “사례”가 도식적으로 더 보여야 한다.
  - 현재: Use Cases는 어두운 카드 3개로만 구성된다.
  - 문제: 사용자가 “건설 안전지도 자동화가 실제로 어떻게 되는지”를 바로 떠올리기 어렵다.
  - 개선: 최소 1개 대표 사례를 도식화한다.
    - `현장 사진/점검표 → AI 정리 → 안전지도 보고서 → 안전보건 대장 업데이트`

- [ ] Products 페이지는 모듈 설명은 좋지만 “도입 후 화면” 상상이 약하다.
  - 현재: 시스템 맵과 모듈 카드는 있으나 실제 산출물의 형태가 적다.
  - 개선: `자동 생성 보고서`, `관리자 대시보드`, `ERP 반영 로그` 같은 산출물 mock을 추가한다.

- [ ] Company 페이지는 신뢰 근거가 부족하다.
  - 현재: “법인 기술 파트너”라는 메시지는 있으나 법인으로서의 안정감 요소가 적다.
  - 개선: 임시라도 아래 블록을 넣을 수 있다.
    - `법인 사업자`
    - `방문 진단 가능`
    - `보안/권한 설계`
    - `기존 시스템 유지`

- [ ] MVP 페이지는 좋은데, MVP 범위/기간/산출물 기대치가 더 명확해야 한다.
  - 현재: 프로세스와 산출물은 있으나 사용자가 “어느 정도 규모인지” 판단하기 어렵다.
  - 개선: `1개 업무`, `샘플 데이터 기준`, `데모/프로토타입`, `본 구현 제안` 같은 경계 문구를 시각적으로 강조한다.

- [ ] Contact 페이지는 문의폼이 좋지만, 실제 접수 전 상태가 드러난다.
  - 현재: 제출 시 “실제 접수 연동은 다음 단계에서 연결” 문구가 나온다.
  - 문제: 외부 공개 시 신뢰를 깎을 수 있다.
  - 개선: 배포 전에는 폼 연동 또는 대체 문구가 필요하다.
    - 예: `문의가 접수되었습니다. 담당자가 확인 후 연락드리겠습니다.`

## P2. 디테일 개선

- [ ] 헤더에 현재 페이지 active 상태가 없다.
  - 개선: 현재 라우트에 해당하는 메뉴에 밑줄, 배경, 컬러 강조를 준다.

- [ ] 모바일 메뉴 열린 상태를 별도로 시각 점검해야 한다.
  - 기본 모바일 화면은 정상이나, 메뉴 오픈 상태의 위치/높이/그림자/터치 영역을 추가 확인해야 한다.

- [ ] 히어로 오른쪽 격자 배경이 모든 페이지에서 반복되어 장식처럼 느껴진다.
  - 개선: 제품/회사/MVP 페이지별로 배경 패턴의 밀도나 위치를 조금씩 바꾸거나, 일부 페이지에서는 제거한다.

- [ ] 버튼 너비 정책이 모바일에서 과하게 통일된다.
  - 현재: 모바일 버튼은 모두 100% 너비다.
  - 문제: 보조 버튼까지 동일하게 무거워 보인다.
  - 개선: 주 CTA만 full width, 보조 CTA는 얇은 텍스트 버튼 또는 작은 outline 버튼으로 낮춘다.

- [ ] 영어 eyebrow와 한글 제목의 톤이 약간 분리되어 보인다.
  - 현재: `Enterprise AI Automation Partner`, `Products`, `Company`, `MVP` 등이 반복된다.
  - 개선: 일부는 한국어로 바꾸거나 더 짧게 정리한다.
    - `자동화 파트너`
    - `제품`
    - `도입 방식`

- [ ] 푸터가 아직 임시 정보처럼 보인다.
  - 개선: 아직 정보가 없다면 푸터를 더 작고 조용하게 만들거나, 입력 예정 문구를 숨긴다.

## 페이지별 수정 체크

### `/v3`

- [ ] 히어로 오른쪽 비주얼을 실제 자동화 구조도로 교체
- [ ] 신뢰 로고 영역을 placeholder처럼 보이지 않게 재구성
- [ ] Problem 카드 6개를 더 명확한 업무 병목 도식으로 변경
- [ ] Use Cases 3개 중 건설 안전지도 사례 하나는 상세 흐름도로 확장
- [ ] 역량 카드의 빈 공간 축소

### `/v3/products`

- [ ] 제품 히어로 비주얼을 모듈 아키텍처 전용으로 변경
- [ ] System Map 섹션의 입력/코어/출력 카드 높이 균형 재조정
- [ ] 제품 모듈 카드 4개에 산출물 예시 추가
- [ ] 산업 패키지 카드에 `도입 전 → 도입 후` 비교 추가
- [ ] 작동 방식 섹션을 단순 카드가 아니라 가로 플로우 라인으로 개선

### `/v3/company`

- [ ] 히어로 제목 축약
- [ ] 법인 신뢰 요소 블록 추가
- [ ] Work Style 카드의 빈 공간 축소
- [ ] Capabilities 카드에 구체 구현 예시 추가
- [ ] CTA를 `방문 진단 요청` 성격으로 변경

### `/v3/mvp`

- [ ] MVP 히어로를 퍼널/검증 흐름 비주얼로 변경
- [ ] `이런 상황이라면` 카드 높이 줄이기
- [ ] MVP 대상 업무를 산업별로 묶기
- [ ] 산출물 섹션에 예시 화면 mock 추가
- [ ] 폼 위에 MVP 진행 범위 요약 추가

### `/v3/contact`

- [ ] 히어로 비주얼을 줄이고 문의폼 접근을 더 빠르게 만들기
- [ ] 문의폼 제출 성공 문구를 실제 서비스 문구처럼 변경
- [ ] 데스크톱에서 좌측 설명과 우측 폼의 세로 정렬 재점검
- [ ] 모바일 폼은 현재 안정적이므로 간격만 약간 압축
- [ ] `문의 전 알아두면 좋은 것` 카드는 더 작게 조정
