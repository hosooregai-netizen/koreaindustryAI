# 공통 UI 구성

## 글씨체 
-  'Lato', 'Noto Sans KR'

## Top bar

### 좌측

- 로고
- 링크: `/v3`
- 의도: v3 메인 페이지로 이동한다.

### 중앙

- Product
- Industries
- Resources
- Company
- 의도: 사이트의 큰 정보 구조를 드롭다운으로 보여준다.

### 우측

- CTA: 문의하기
- 링크: `/v3/contact`
- 의도: 모든 페이지에서 상담 전환을 고정한다.

### 배경

- 최상단 위치시, 투명 배경으로 바뀌며, 히어로 섹션영역이 배경으로 나타나도록
- 기본: 흰색 배경.
- 최상단 히어로 위에서도 같은 높이와 크기를 유지한다.

### 컴포넌트

- 좌측 로고: 기존 로고 유지.
- 중앙 메뉴: 검정 글씨, 얇고 깔끔한 인상.
- 우측 CTA: 투명 배경, 글씨와 같은 색의 각진 스트로크.
- 모바일: 메뉴 버튼으로 열고, 드롭다운 항목은 세로로 모두 보이게 한다.

### 드롭다운

- 적용 대상: 중앙 메뉴만.
- 중앙 메뉴에서 hover 또는 focus 시 테두리가 둥근 네모 패널을 띄운다.
- 패널은 현재보다 넓게 사용한다.
- 구성 
    -좌: 대표 이미지 1개, 그룹 설명 1문장
    -우: 세부 페이지 제목 목록.
- 대표 이미지는 섹션을 상징하는 이미지로 사용한다.
- 설명은 아주 짧게 쓰고, 페이지별 설명은 화면에 노출하지 않는다.
- 실제 링크가 있는 항목은 페이지로 이동한다.
- 준비중 항목도 준비중 stub 라우트로 이동한다.
- 검증: hover, focus, 모바일 터치, 링크 이동.

## 카드 형태

### 기본 카드

- 용도: 반복 항목, 기능 목록, FAQ 보조 정보.
- 구성: 짧은 제목, 1문장 설명, 필요 시 보조 링크.
- 제한: 카드 안에 카드를 넣지 않는다, 뱃지를 달지 않는다.
- 제목 문구 제한: 핵심 기능만 명사형으로 넣는다.
- 설명 문구 제한: 핵심 기능에 대한 설명만 짧고 명확하게 적는다.
- 디자인: 테두리 끝이 살짝 둥근 형태.
- 동작: hover 시 살짝 커지는 동적 구성.

### 이미지 카드

- 용도: 이미지가 들어가는 카드.
- 구성: 이미지가 가득 찬 카드.
- 디자인: 테두리 끝이 살짝 둥근 형태.
- 동작: hover 시 살짝 커지는 동적 구성.

## 글씨 크기 및 두께 정책

- 기준 폰트: `Lato`, `Noto Sans KR`
- 기본 본문: 16px / line-height 1.6 / font-weight 400
- 보조 설명: 14px / line-height 1.55 / font-weight 400
- 탑바 업체명: 16px / line-height 1 / font-weight 700
- 탑바 메뉴: 15px / line-height 1 / font-weight 400
- 탑바 CTA: 15px / line-height 1 / font-weight 400
- 드롭다운 설명: 14px / line-height 1.55 / font-weight 400
- 드롭다운 항목 제목: 15px / line-height 1.35 / font-weight 400
- 카드 제목: 22px / line-height 1.3 / font-weight 700
- 카드 설명: 15px / line-height 1.6 / font-weight 400
- 섹션 제목: 40px / line-height 1.15 / font-weight 700
- 히어로 제목: 64px / line-height 1.05 / font-weight 700
- 모바일 히어로 제목: 42px / line-height 1.1 / font-weight 700

### 볼드체 사용 기준

- `font-weight 400`: 기본값. 탑바 메뉴, CTA, 설명, 본문, 드롭다운 항목에 사용한다.
- `font-weight 500`: 숫자, 짧은 라벨, 작은 강조가 필요할 때만 제한적으로 사용한다.
- `font-weight 700`: 업체명, 카드 제목, 섹션 제목, 히어로 제목처럼 구조를 구분하는 제목에만 사용한다.
- `font-weight 800 이상`: 기본 UI에는 사용하지 않는다. 아주 짧은 캠페인성 키워드나 특수 강조가 필요할 때만 예외적으로 사용한다.
- 같은 컴포넌트 안에서는 2개 이상의 두께를 섞지 않는다.
- hover, focus, active 상태에서 글씨 두께를 바꾸지 않는다. 상태 변화는 색상, 밑줄, 배경, 투명도로 처리한다.
