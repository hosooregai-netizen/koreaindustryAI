# 대한산업AI site 페이지 기획 작업 문서

## 1. 공통 작업 정책

- 관련 문서를 먼저 확인한다: 웹 구성은 `web.md`, UI는 `ui.md`, 페이지 내용은 `page-detail.md`.
- 없는 페이지는 실제 링크로 연결하지 않는다.
- 탑바에 노출하는 준비중 항목은 실제 준비중 stub 라우트로 이동한다.
- 모든 CTA는 라벨, 링크 또는 클릭 후 동작을 명확히 적는다.
- 모든 페이지와 메뉴 항목은 `구현`, `준비중`, `보류` 중 하나의 상태를 가진다.
- 실제 고객명, 성과 수치, 제공 기능은 검증 전까지 확정된 사실처럼 쓰지 않는다.
- 공통 UI 변경은 `ui.md` 기준을 우선한다.
- 페이지별 섹션 변경은 `page-detail.md` 기준을 우선한다.
- 모바일에서 메뉴, CTA, 폼을 확인하기 전에는 완료로 보지 않는다.
- 구현 후에는 문서와 실제 코드가 같은 구조를 말해야 한다.

## 2. 전체 페이지 구조

### 메인 페이지

- 상태: 구현
- 라우트: `/`
- 목적: 첫인상, 핵심 메시지, 신뢰, 문의 전환.
- 완료 기준: 첫 화면에서 대한산업AI가 하는 일과 문의 경로가 보인다.
- 히어로 기준: 2개 영상 시퀀스와 1600px 기준 왼쪽 하단 본문 정렬을 유지한다.

### Product

- 상태: 일부 구현
- 목적: 제공 제품과 작동 방식을 설명한다.

#### Data-Driven AI-Core

- 라우트: `/products/data-driven`
- 상태: 구현

- 주의: 탑바 라벨과 본문 제품명은 `Data-Driven AI-Core`로 통일한다.

#### Automation AI-Core

- 라우트: `/products/automation`

- 의도: 큰 구축 전 작은 반복 업무 자동화로 시작하는 준비중 제품.
- 현재 처리: 클릭 시 준비중 stub 페이지로 이동.
