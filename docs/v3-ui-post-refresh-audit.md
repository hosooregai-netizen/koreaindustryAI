# v3 UI 후속 점검 체크리스트

점검 기준: `/v3`, `/v3/products`, `/v3/company`, `/v3/mvp`, `/v3/contact` 데스크톱 1440px, 모바일 390px 화면을 직접 캡처해서 확인했습니다. 캡처 중 좌하단에 보이는 `N` 배지는 Next.js 개발 서버 표시로, 배포 화면에서는 제외됩니다.

## 우선 수정 권장

- [ ] 홈 히어로 H1이 데스크톱에서 너무 길게 가로로 뻗어 모니터 영역까지 침범하는 느낌이 있음. 최대 폭을 조금 줄이거나 폰트 크기를 낮춰 좌측 카피 블록으로 더 단단하게 묶는 편이 좋음.
- [ ] 모바일 홈 히어로에서 제목 줄바꿈은 읽히지만 세로 공간이 빽빽함. H1 크기, CTA 간격, 진행바 간격을 조금 줄이면 안정적임.
- [ ] 제품 쇼케이스의 큰 문구가 섹션 내부치고 너무 커서 “두 번째 히어로”처럼 보임. 본문 섹션 제목은 한 단계 낮추고 이미지와 수평 균형을 맞추는 편이 좋음.
- [ ] 산업 카드와 스토리 카드가 같은 히어로 이미지를 반복 사용해 실제 사례/산업별 차이가 약하게 느껴짐. 건설, 제조, ERP, 문서 자동화별로 다른 이미지나 다른 크롭이 필요함.
- [ ] 자동화 레이어 도식은 정보는 명확하지만 연결선/방향성이 약함. 입력 → AI 처리 → 산출물 흐름이 더 도식적으로 보이도록 선, 화살표, 단계 번호를 강화할 필요가 있음.

## 데스크톱 밸런스

- [ ] 서브페이지 히어로의 우측 mockup이 모든 페이지에서 거의 동일해 페이지별 개성이 약함. 제품/회사/MVP/문의별 아이콘, 상태값, 화면 구성을 조금씩 다르게 보이게 하면 좋음.
- [ ] 탑바 active 색상이 파란색이라 레퍼런스의 블랙/화이트 고급 톤보다 SaaS 느낌이 강함. active는 검정 굵기, 짧은 밑줄, 혹은 미세한 opacity 차이 정도로 정리 가능.
- [ ] 로고 스트립은 텍스트 로고처럼 보여 신뢰 로고 영역보다는 임시 텍스트 느낌이 있음. 실제 로고 자산이 없으면 더 연한 워드마크 스타일이나 “적용 가능 산업” 형태로 바꾸는 편이 자연스러움.
- [ ] 문의 페이지 2번째 섹션은 좌측 설명 아래가 많이 비어 있음. 문의 전 확인사항, 상담 방식, 예상 산출물 같은 보조 정보를 넣거나 form과 높이를 맞추면 균형이 좋아짐.
- [ ] 제품 페이지의 “작동 방식” 단계 카드가 너무 단순해서 하단 검은 CTA와 비교해 힘이 빠짐. 단계별 짧은 설명 또는 아이콘을 추가하면 흐름이 좋아짐.

## 모바일

- [ ] 모바일 홈 로고 스트립에서 회사명이 세로로 길게 쌓여 첫 화면 이후 흐름이 늘어짐. 모바일에서는 2열 그리드, 가로 스크롤, 또는 3개만 노출하는 방식이 적합함.
- [ ] 모바일 제품 페이지가 전체적으로 길고 카드가 연속되면서 리듬이 단조로움. 주요 섹션 사이에 이미지형 구간 또는 요약 CTA를 넣으면 피로도가 줄어듦.
- [ ] 모바일 문의 페이지의 “문의 때 알려주면 좋은 것” 카드가 텍스트 대비 높이가 커서 빈 카드처럼 보임. 카드 최소 높이를 낮추거나 체크 아이콘/짧은 설명을 추가하면 좋음.
- [ ] 모바일 회사소개의 기능 카드가 세로로 길게 반복되어 정보 밀도가 낮아짐. 2개씩 묶거나 accordion/compact list 형태가 더 적합할 수 있음.
- [ ] 모바일 히어로 mockup은 예쁘지만 본문보다 아래로 밀려 CTA 이후 흐름이 길어짐. 일부 페이지에서는 mockup을 더 작게 하거나 첫 화면에서 숨기는 것도 고려 가능.

## 콘텐츠/신뢰감

- [ ] “Samsung, SK hynix…”는 임시 고객 로고로 보이는데 실제 고객이 아니라면 리스크가 있음. “예상 적용 산업” 또는 “적용 가능 시스템”으로 표현을 바꾸는 편이 안전함.
- [ ] 회사소개에서 법인 신뢰를 주려면 사업자 정보, 대표 서비스 범위, 방문 진단 프로세스, 보안/권한 처리 기준이 더 명확히 보이면 좋음.
- [ ] 제품/사례 문구는 좋아졌지만 실제 산출물 예시가 아직 추상적임. 안전지도 보고서, 안전보건 대장, ERP 반영 로그, 재고 알림 화면을 더 구체적인 미니 시각물로 보여주면 설득력이 커짐.
- [ ] “Daehan Industry AI”, “live preview” 같은 영문 UI 문구가 한국어 랜딩과 살짝 분리되어 보임. 한국어 중심으로 바꾸거나 아주 작은 시스템 라벨로 낮추는 편이 자연스러움.

## 유지해도 좋은 점

- [x] 흰 탑바, 중앙 메뉴, 우측 문의 CTA 구조는 레퍼런스와 비슷한 방향으로 안정적임.
- [x] 홈 히어로의 산업 현장 이미지와 큰 문구 조합은 첫인상에서 신뢰감이 있음.
- [x] 메인 / 제품 / 회사소개 / MVP / 문의 페이지가 각각 독립 페이지처럼 구분됨.
- [x] 문의폼은 데스크톱과 모바일 모두 입력 흐름 자체는 깨지지 않음.
- [x] 전체 톤은 이전보다 훨씬 “기술 업체”에 가까워졌고, 랜딩의 목적도 더 선명해짐.
