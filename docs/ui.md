# 공통 UI 구성

## 글씨체

- `Noto Sans KR`, `Lato`
- 한글 문장 안에 영어가 섞이는 본문, 설명, 메뉴는 `Noto Sans KR`을 우선 사용한다.
- 순수 영문 로고나 짧은 영문 강조가 필요할 때만 `Lato`를 보조로 사용한다.

## Top bar

### 좌측

- 로고
- 링크: `/`
- 의도: site 메인 페이지로 이동한다.

### 중앙

- Product
- Industries
- Community
- Company
- 의도: 사이트의 큰 정보 구조를 드롭다운으로 보여준다.

### 우측

- CTA: 문의하기
- 링크: `/contact`
- 의도: 모든 페이지에서 상담 전환을 고정한다.

### 배경

- 최상단 위치시, 투명 배경으로 바뀌며, 히어로 섹션영역이 배경으로 나타나도록
- 1묶음 히어로 구간: 투명 배경 + 브랜드, 로고, nav, 문의하기 버튼 전체 `#ffffff`.
- 2묶음 히어로 구간: 투명 배경 + 브랜드, 로고, nav, 문의하기 버튼 전체 `#000000`.
- 스크롤 후: 흰 배경 + 검정 글씨를 유지한다.
- 기본: 흰색 배경.
- 최상단 히어로 위에서도 같은 높이와 크기를 유지한다.

### 컴포넌트

- 좌측 로고: 기존 로고 유지.
- 중앙 메뉴: 검정 글씨, 얇고 깔끔한 인상.
- 우측 CTA: 투명 배경, 글씨와 같은 색의 각진 스트로크.
- 모바일: 메뉴 버튼으로 열고, 드롭다운 항목은 세로로 모두 보이게 한다.

## Hero 영상

- 영상 소스: 1묶음은 `/assets/hero-landing-intro.mp4`, 2묶음은 `/assets/hero-landing-reverse.mp4`를 사용한다.
- 로딩 정책: 첫 로딩에서 썸네일만 보이지 않도록 실제 재생 가능 상태 기준으로 전환한다.
- 전환 정책: 영상 전환 시 이전 영상의 마지막 프레임을 캡처한 freeze 레이어와 다음 영상 레이어를 함께 사용한다.
- 본문 위치: 홈 히어로 텍스트는 1600px 기준 콘텐츠 박스의 왼쪽 하단에 배치하고, 작은 화면에서는 24px 이상 여백을 유지한다.
- 본문 텍스트: 1묶음은 `#ffffff`, 2묶음은 `#000000`을 사용한다.
- 묶음 전환 효과: 영상이 살짝 축소된 뒤 화면 전체가 왼쪽으로 넘어가는 전환 효과를 사용한다.
- 텍스트 전환: 묶음 전환 시 텍스트는 fade 전환을 사용한다.

### 드롭다운

- 적용 대상: 중앙 메뉴만.
- 중앙 메뉴에서 hover 또는 focus 시 테두리가 둥근 네모 패널을 띄운다.
- 패널은 현재보다 넓게 사용한다.
- 구성 
    -좌: 대표 이미지 1개, 그룹 설명 1문장
    -우: 세부 페이지 제목 목록.
- 대표 이미지는 섹션을 상징하는 이미지로 사용한다.
- 설명은 아주 짧게 쓰고, 페이지별 설명은 화면에 노출하지 않는다.
- 실제 링크가 있는 항목은 페이지로 이동한다.
- 준비중 항목도 준비중 stub 라우트로 이동한다.
- 검증: hover, focus, 모바일 터치, 링크 이동.

## 카드 형태

### 기본 카드

- 용도: 반복 항목, 기능 목록, FAQ 보조 정보.
- 구성: 짧은 제목, 1문장 설명, 필요 시 보조 링크.
- 제한: 카드 안에 카드를 넣지 않는다, 뱃지를 달지 않는다.
- 제목 문구 제한: 핵심 기능만 명사형으로 넣는다.
- 설명 문구 제한: 핵심 기능에 대한 설명만 짧고 명확하게 적는다.
- 디자인: 테두리 끝이 살짝 둥근 형태.
- 동작: hover 시 살짝 커지는 동적 구성.

### 이미지 카드

- 용도: 이미지가 들어가는 카드.
- 구성: 이미지가 가득 찬 카드.
- 디자인: 테두리 끝이 살짝 둥근 형태.
- 동작: hover/focus 시 살짝 커지는 동적 구성.
- 산업 이미지 카드와 What's New 이미지 카드: hover/focus 시 오른쪽에 검은 그라데이션을 얹고, 배경이나 테두리 없는 단색 선형 화살표를 표시해 이동 가능성을 보여준다. 화살표는 텍스트 라벨이 아니라 시각 마크로만 처리한다.

### Solution 카드 섹션

- 용도: 홈의 신뢰 로고와 Products 사이에서 대한산업AI의 솔루션 설계 방식을 설명한다.
- 레퍼런스: Mondrian AI Infra 페이지처럼 상단 중앙 카피와 3카드 구조를 참고하되, 중간에 흐르는 대형 문자 연출과 `Overview` 헤딩은 사용하지 않는다.
- 배경: Products와 이어지도록 흰색 배경을 유지한다.
- 구조: 상단 중앙 제목/설명, 큰 여백, 하단 3개 카드.
- 카드: 8px 이하 radius의 다크 네이비 카드로 만들고, 카드 내부에 푸른 곡선 라인 장식을 배치한다.
- 장식 제한: 장식은 카드 내부의 선형 아크로만 처리하고, 섹션 배경에 별도 오브젝트나 흐르는 텍스트를 두지 않는다.
- 콘텐츠: 카드 번호, 짧은 제목, 한 문장 설명만 넣는다. 카드 안에 버튼, 링크, 뱃지, 중첩 카드를 넣지 않는다.
- 동작: 스크롤 재진입 시 상단 카피와 카드가 순서대로 fade-up 된다.
- hover: 클릭 가능한 요소처럼 보이지 않게 색상 반전이나 커서 변화는 피하고, 카드가 6px 이하로만 미세하게 떠오르게 한다.
- 반응형: 데스크톱은 3열, 980px 이하는 1열로 전환한다. 모바일에서는 텍스트가 카드 밖으로 넘치지 않도록 자연스러운 줄바꿈을 허용한다.

### Products 비주얼 배경

- 용도: Products 섹션의 제품 UI 스크린샷 뒤에 들어가는 비주얼 배경.
- 자산: `/assets/배경2.png`.
- 톤: 솔루션 카드와 이어지도록 딥 네이비, 코발트 블루, 절제된 퍼플 계열을 사용한다.
- 처리: UI 스크린샷의 가독성을 위해 어두운 오버레이를 얇게 얹고, 흰색 워시나 밝은 산업 이미지 배경은 사용하지 않는다.
- 동작: hover/focus 시 배경 이미지는 1.035배 이하로만 확대하고, 채도와 대비를 아주 약하게 올린다.
- 행 hover: 제품 행 전체 hover 색은 초록 계열로 분기하지 않고, 두 제품 모두 차가운 블루 톤의 매우 옅은 배경으로 통일한다.

## 글씨 크기 및 두께 정책

- 기준 폰트: `Noto Sans KR`, `Lato`
- 기본 본문: 16px / line-height 1.6 / font-weight 400
- 보조 설명: 14px / line-height 1.55 / font-weight 400
- 탑바 업체명: 18px / line-height 1 / font-weight 700
- 탑바 메뉴: 16px / line-height 32px / font-weight 400
- 탑바 CTA: 16px / line-height 1.6 / font-weight 400
- 드롭다운 설명: 14px / line-height 1.55 / font-weight 400
- 드롭다운 항목 제목: 15px / line-height 1.35 / font-weight 400
- 카드 제목: 22px / line-height 1.3 / font-weight 700
- 카드 설명: 15px / line-height 1.6 / font-weight 400
- 섹션 제목: 40px / line-height 1.4 / font-weight 500
- 제품/산업 섹션 타이틀: 56px / line-height 1 / font-weight 600
- 제품명 타이틀: 56px / line-height 1.05 / font-weight 600
- 히어로 제목: 64px / line-height 1.3 / font-weight 700
- 히어로 설명문: 20px / line-height 1.5 / font-weight 400
- 모바일 히어로 제목: 42px / line-height 1.3 / font-weight 700

### 볼드체 사용 기준

- `font-weight 400`: 기본값. 탑바 메뉴, CTA, 설명, 본문, 드롭다운 항목에 사용한다.
- `font-weight 500`: 섹션 제목, 숫자, 짧은 라벨, 작은 강조가 필요할 때만 제한적으로 사용한다.
- `font-weight 700`: 업체명, 카드 제목, 히어로 제목처럼 구조를 강하게 구분하는 제목에만 사용한다.
- `font-weight 800 이상`: 기본 UI에는 사용하지 않는다. 아주 짧은 캠페인성 키워드나 특수 강조가 필요할 때만 예외적으로 사용한다.
- 같은 컴포넌트 안에서는 2개 이상의 두께를 섞지 않는다.
- hover, focus, active 상태에서 글씨 두께를 바꾸지 않는다. 상태 변화는 색상, 밑줄, 배경, 투명도로 처리한다.
