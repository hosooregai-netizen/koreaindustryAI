- 영상 묶음: 1번 영상은 기존 1~3장면을 합친 1묶음으로 약 13초, 2번 영상은 `/assets/hero-landing-reverse.mp4` 기준 약 12초를 사용한다.
- 본문 위치: 데스크톱 홈 히어로 본문은 전체 화면 왼쪽이 아니라 1600px 콘텐츠 기준선의 왼쪽 하단에 배치한다. 1600px 이하에서는 최소 24px 좌우 여백을 유지한다.

- 2묶음 히어로 제목: `반복 업무를 자동으로 처리하는 AI 시스템`

- 영상 정책: 모든 영상은 워터마크가 보이지 않도록 크롭 처리된 버전을 사용한다.
- 검증: 모바일 첫 화면에서 제목, 설명, 진행 컨트롤이 겹치지 않는다.

- 별도 헤딩: 없음.

- 스크롤로 섹션이 보일 때 상단 제목과 설명이 먼저 fade-up 된다.

- 3개 카드는 좌측부터 순차 fade-up 된다.
- 카드 hover 시 색상 반전이나 CTA처럼 보이는 변화는 주지 않고, 6px 이하의 미세한 상승과 그림자 변화만 사용한다.
- 브릿지 섹션 안에는 버튼, 카드 CTA, 추가 링크를 넣지 않는다.

# 2. 페이지별 구성 - Industry

## 목적

- 산업별 업무 맥락을 같은 페이지 구조 안에서 다르게 보여준다.
- 실제 고객 사례처럼 보이지 않게 하고, 명함 문구 기준의 적용 가능 산업과 업무 영역으로 표현한다.
- 홈의 산업 카드에서 진입했을 때 히어로, Overview, 산업별 특징, 최종 CTA, 푸터로 빠르게 읽히는 상세 페이지를 만든다.

## 라우트

- 제조: `/industries/manufacturing`
- 물류: `/industries/logistics`
- 유통: `/industries/distribution`
- 건설: `/industries/construction`
- 금융: `/industries/finance`
- 현재 코드에 없는 `유통` 라우트는 새로 추가한다. 기존 홈의 `물류` 카드와 분리할지, `물류/유통` 묶음으로 운영할지는 구현 전에 최종 결정한다.

## 공통 페이지 구조

1. Hero
2. Overview
3. 산업별 특징
4. 최종 CTA
5. Footer

## Hero

- 참고 이미지: 이미지 1.
- 목적: 방문자가 첫 화면에서 어떤 산업 페이지에 들어왔는지 즉시 이해하게 한다.
- 구성: 산업별 이미지를 좌우 끝까지 채우는 가로 full-bleed 이미지 히어로로 구성한다.
- 높이: 화면 전체 커버가 아니다. 데스크톱은 약 420~520px, 태블릿은 약 360~420px, 모바일은 약 300~360px 범위에서 안정적으로 보이게 한다.
- 이미지 처리: `object-fit: cover`로 좌우를 채우되, 핵심 피사체가 잘리지 않도록 산업별 `object-position`을 조정한다.
- 오버레이: 이미지 위에 어두운 오버레이를 얹어 흰색 텍스트 가독성을 확보한다. 오버레이는 과하게 검게 만들지 않고 산업 이미지가 읽히게 둔다.
- 텍스트 정렬: 이미지 1처럼 중앙 정렬한다.
- 최대 폭: 텍스트는 약 960px 안에서 정리한다.
- CTA: 히어로에는 버튼을 넣지 않는다. 전환 행동은 최종 CTA에서 처리한다.
- 라벨: `AI 기반 산업 운영 자동화`
- 제목 형식: `{{산업명}} 운영 자동화`

## Data-Driven AI-Core 페이지

### Hero

- 목적: 현장 데이터와 문서, 승인 흐름을 기업 맞춤 ERP 구조로 연결한다는 메시지를 첫 화면에서 명확하게 전달한다.
- 방향: 참고 이미지처럼 밝은 블루 계열 배경 위에 중앙 정렬 히어로를 구성한다. 과한 제품 스크린샷이나 복잡한 장식보다, 제품명이 아니라 고객이 얻는 업무 결과가 먼저 읽히게 한다.
- 배경: 상단은 흰색에 가깝고 중앙으로 갈수록 옅은 블루가 깔리는 부드러운 그라데이션을 사용한다.
- 배경 패턴: 히어로 전체에 아주 옅은 도트 그리드 패턴을 넣는다. 패턴은 텍스트보다 튀지 않게 `opacity`를 낮추고, 모바일에서도 글자 뒤가 지저분해 보이지 않아야 한다.
- 높이: 데스크톱 기준 첫 화면 대부분을 차지하되 다음 섹션의 시작이 살짝 보이게 한다. 모바일에서는 불필요하게 긴 여백 없이 제목과 CTA가 첫 화면 안에 안정적으로 들어온다.
- 정렬: 모든 텍스트와 CTA는 중앙 정렬한다.
- 최대 폭: 텍스트 영역은 약 760px 안에서 정리한다.
- 라벨: `Data-Driven AI-Core`
- 제목: `현장 데이터 걱정 없이 AI 업무 시작`
- 설명: `문서, 승인 흐름, 운영 데이터를 기업 맞춤 ERP 구조로 연결해 업무 화면과 보고서로 바로 활용할 수 있게 합니다.`
- CTA:
  - Primary: `데이터 구조 상담` / `/contact`
  - Secondary: `도입 방식 보기` / 히어로 아래 `데이터 구성 카드` 섹션 앵커
- CTA 스타일: Primary는 검정 배경의 pill 버튼, Secondary는 흰색 또는 반투명 흰색 배경의 외곽선 pill 버튼을 사용한다.
- 본문 톤: 제목은 검정에 가까운 색, 설명은 짙은 네이비 또는 회색 계열을 사용한다.
- 시각 요소: 히어로 안에는 별도 카드형 제품 UI를 크게 넣지 않는다. 필요하면 하단 가장자리에 아주 희미한 데이터 흐름 라인이나 도트 확산 정도만 배치한다.
- 동작: 진입 시 라벨, 제목, 설명, CTA가 순서대로 짧게 fade-up 된다.
- 검증: 제품명보다 `현장 데이터`, `업무 화면`, `보고서`, `활용` 같은 결과 표현이 먼저 읽힌다.

### 내부 구성 원칙

- 방향: 참고 이미지처럼 밝은 배경, 큰 여백, 중앙 정렬 섹션 타이틀, 라운드가 낮은 대형 비주얼 카드로 구성한다.

- 언어: 제품명(`Data-Driven AI-Core`)을 제외한 UI 내부 문구는 한국어를 우선 사용한다.

- 동작: 각 카드가 스크롤 진입 시 순차 fade-up 된다.
- hover: 이미지 영역만 미세하게 확대하고, 카드 전체 색상 반전은 사용하지 않는다.

### 최종 CTA와 푸터

- 최종 CTA: 메인 페이지의 최종 CTA 컴포넌트를 재사용한다.
- CTA 문구는 현재 메인 페이지 문구를 우선 그대로 사용한다.
- 푸터: 메인 페이지 푸터를 그대로 재사용한다.

# 4. 페이지별 구성 - Community

## 목적

- Newsletter, Blog, Technology 콘텐츠를 같은 구조 안에서 탐색할 수 있게 한다.
- 홈의 `What's New` 섹션에서 진입한 사용자가 각 콘텐츠 유형의 목록을 바로 확인하게 한다.
- `News` 페이지는 운영하지 않는다. 보도자료성 소식은 `Newsletter` 또는 `Blog` 콘텐츠로 흡수한다.
- 참고 이미지처럼 블로그형 콘텐츠 허브 톤으로 구성한다. 이미지 히어로나 전환 CTA보다 글 목록 자체가 첫 화면의 중심이 되게 한다.

## 라우트

- Newsletter: `/community/newsletter`
- Blog: `/community/blog`
- Technology: `/community/technology`
- 제외: `/community/news`

## 공통 페이지 구조

1. 페이지 인트로
2. 대표 콘텐츠
3. 콘텐츠 그리드
4. Pagination
5. Footer

## 페이지 인트로

- 참고 이미지: 상단 `Blog` 예시.
- 목적: 페이지명과 한 줄 설명만으로 콘텐츠 페이지에 들어왔다는 신호를 준다.
- 구성: 흰색 배경 위에 큰 페이지 제목과 짧은 설명을 좌측 정렬로 배치한다.
- 제외: full-bleed 이미지 히어로, 어두운 오버레이, 히어로 CTA, 구독 모달 진입 버튼은 사용하지 않는다.
- 최대 폭: 1600px 콘텐츠 기준선을 유지한다.
- 상단 여백: 헤더 아래에서 충분히 숨을 두되, 첫 화면에서 대표 콘텐츠 일부가 함께 보이도록 과하게 높이지 않는다.
- 제목: 각 페이지 이름인 `Newsletter`, `Blog`, `Technology`를 사용한다.
- 설명: 각 페이지의 성격을 한 문장으로 정리한다.
- 동작: 진입 시 과한 애니메이션 없이 콘텐츠가 즉시 읽히게 한다. 필요하면 아주 짧은 fade-up만 사용한다.

## 대표 콘텐츠

- 참고 이미지: 상단 큰 글 영역.
- 목적: 최신 또는 가장 중요한 글 1개를 페이지 최상단에서 가장 크게 보여준다.
- 데이터 기준: 현재 페이지에 노출되는 콘텐츠 목록에서 첫 번째 항목을 대표 콘텐츠로 사용한다.
- 구성: 좌측 큰 이미지, 우측 제목/요약/날짜 텍스트의 2열 구조.
- 이미지 비율: 데스크톱에서는 넓은 가로형 이미지를 사용하고, 모서리는 8px radius를 넘기지 않는다.
- 제목: 일반 카드보다 훨씬 크게 표시한다.
- 요약: 대표 콘텐츠에만 1~2줄 요약을 표시한다. 일반 카드에는 요약을 넣지 않는다.
- 태그/뱃지: `Newsletter`, `AI-Core`, `Monthly letter`처럼 반복적으로 보이는 pill 뱃지는 노출하지 않는다.
- 링크: 상세 URL이 확정된 콘텐츠만 대표 영역 전체 링크를 활성화한다. 임시 콘텐츠는 링크 없이 표시한다.
- 반응형: 모바일에서는 이미지와 텍스트가 1열로 쌓이고, 제목은 화면 폭에 맞게 줄바꿈한다.

## 콘텐츠 그리드

- 참고 이미지: 하단 4열 카드 그리드.
- 목적: 대표 콘텐츠 아래에 나머지 글을 빠르게 스캔할 수 있게 보여준다.
- 필터링: 첫 구현에서는 필터와 검색을 넣지 않는다. 콘텐츠가 충분히 쌓인 뒤 필요하면 별도 단계에서 추가한다.
- 구성: 각 카드에는 이미지, 제목을 표시한다. 날짜는 필요하면 제목 아래 보조 정보로 작게 둔다.
- 카드 이미지: 16:9 또는 그에 가까운 가로형 비율을 유지한다.
- 카드 스타일: 카드 자체를 무거운 박스로 감싸지 않고, 이미지와 텍스트가 자연스럽게 놓이는 블로그 목록 톤을 사용한다.
- 태그/뱃지: 첫 구현에서는 카드 태그와 카테고리 뱃지를 사용하지 않는다. 콘텐츠가 늘어나 분류가 꼭 필요해질 때 별도 단계에서 다시 검토한다.
- 동작: hover/focus 시 이미지만 미세하게 확대하거나 그림자를 아주 낮게 준다. CTA처럼 보이는 화살표 오버레이는 사용하지 않는다.
- 배치: 데스크톱은 4열, 좁은 데스크톱/태블릿은 2열, 모바일은 1열로 접는다.
- 최대 노출: 한 페이지에서 대표 콘텐츠 1개와 일반 카드 최대 8개, 총 9개까지 노출한다.
- 임시 데이터 정책: 문서에는 임시 콘텐츠 목록을 적지 않는다. 구현 단계에서 레이아웃 확인용으로 각 페이지별 2~3개 정도만 임시 생성한다.

## Pagination

- 목적: 콘텐츠 수가 늘어나도 목록 영역이 과하게 길어지지 않게 한다.
- 페이지당 최대 노출 수: 대표 콘텐츠 1개 + 일반 카드 8개, 총 9개.
- 배치: 콘텐츠 카드 그리드 하단 중앙에 이전, 페이지 번호, 다음 컨트롤을 표시한다.
- 표시 조건: 전체 콘텐츠가 10개 이상일 때 pagination을 표시한다.
- 비활성 상태: 첫 페이지에서는 이전 버튼, 마지막 페이지에서는 다음 버튼을 비활성화한다.
- 페이지 전환: 페이지가 바뀌면 해당 페이지 묶음의 첫 번째 콘텐츠가 다시 대표 콘텐츠로 크게 표시된다.
- 접근성: 현재 페이지는 `aria-current="page"`로 표시하고, 이전/다음 버튼에는 명확한 `aria-label`을 제공한다.

# 5. 페이지별 구성 - Company

## 목적

- 회사 신뢰와 일하는 방식을 설명한다.

## 회사 소개

### Hero

- 대한산업AI가 어떤 기술 파트너인지 설명한다.

### 작업 방식

- 현장 또는 원격 미팅.
- 업무 화면 확인.
- 문서 양식 분석.
- 시현 범위 제안.

### CTA

- 문의하기.

## 고객 문의

### 레이아웃 방향

- 참고 이미지처럼 첫 화면을 `좌측 안내 영역 + 우측 문의 폼 패널`의 2열 split 구조로 구성한다.
- 기존 큰 이미지 Hero가 먼저 보이는 구조는 사용하지 않는다. 문의 페이지의 첫 신호는 곧바로 `문의하기`여야 한다.
- 전체 배경은 흰색, 좌우 여백은 넓게 두고 최대 폭은 1600px 기준으로 정리한다.
- 좌측 영역은 넓은 빈 공간을 살려 간결한 문구와 상담 진행 단계를 보여준다.
- 우측 영역은 연한 회색 폼 패널로 묶는다. 패널은 과한 장식 없이 큰 여백, 명확한 입력 필드, 낮은 대비의 보더만 사용한다.
- 패널 모서리는 사이트 카드 규칙에 맞춰 8px 이내로 유지한다.

### 좌측 카피

- H1: `산업 AI 전문가에게 문의하기`
- 본문:
  - `반복 보고, 승인 흐름, 운영 데이터처럼`
  - `매일 붙잡고 있는 업무를 남겨주세요.`
  - `대한산업AI가 현장 맥락을 확인하고 AI-Core로 연결할 수 있는`
  - `데이터 구조와 자동화 범위를 정리해드립니다.`
- 보조 문구:
  - `상담 전 자료가 완벽하지 않아도 괜찮습니다. 현재 쓰는 문서, 시스템, 반복 업무만 편하게 적어주세요.`
  - 파란 세로선처럼 강한 장식은 사용하지 않는다. 흰색 바탕과 낮은 대비의 보더로 안내 문구를 조용히 묶는다.
- 하단 진행 단계:
  - `문의 접수`
  - `내용 검토`
  - `상담 일정 안내`
  - `맞춤형 제안`
- 진행 단계는 본문과 지나치게 멀리 떨어뜨리지 않는다. 보조 문구 바로 아래에 낮은 대비의 구분선과 작은 chevron으로 연결된 프로세스처럼 보이게 한다.
- 별도 카드나 강한 배경으로 만들지 않는다.

### 문의 폼 패널

- 상단 안내 문구:
  - `지금 문의를 남겨주세요.`
  - `업무 맥락을 확인한 뒤 적합한 AI-Core 구성과 상담 일정을 안내드리겠습니다.`
- 필드:
  - `성함 *`
  - `회사명 *`
  - `업무용 이메일주소 *`
  - `휴대폰번호 *`
  - `대한산업AI를 어떻게 알게 되셨나요?`
  - `도입 검토 상황을 알려주세요.`
  - `어떤 솔루션에 관심이 있으신가요? *`
  - `문의내용 *`
- select placeholder는 `선택해 주세요.`로 통일한다.
- 문의내용 placeholder:
  - `예: 매주 작성하는 보고서, 승인 흐름, 운영 데이터 정리처럼 자동화하고 싶은 업무를 적어주세요.`
- 버튼 라벨: `문의 접수하기`
- 성공 메시지: `문의가 접수되었습니다. 담당자가 확인 후 연락드리겠습니다.`
- 기본 안내 메시지: `남겨주신 정보는 상담 검토와 회신 목적으로만 사용합니다.`

### 폼 UI 규칙

- 입력 필드는 흰색 pill 형태로 보이되, 높이와 좌우 패딩을 충분히 둔다.
- label은 필드 위에 배치하고, 필수 항목은 `*`로 표시한다.
- 폼 패널 내부 간격은 넉넉하게 잡아 레퍼런스처럼 숨 쉬는 느낌을 만든다.
- 폼 상단 안내는 얇은 구분선으로 필드 영역과 분리한다.
- textarea는 첫 화면 하단에서 자연스럽게 이어지도록 충분한 높이를 둔다.
- CTA 버튼은 폼 하단에 배치한다. 버튼은 사이트 기본 primary 스타일을 사용하되 폼 폭에 맞춰 명확하게 보이게 한다.

### 반응형

- 데스크톱: 좌측 안내 영역과 우측 폼 패널을 2열로 배치한다. 우측 폼 패널이 너무 넓어지지 않도록 최대 폭을 둔다.
- 태블릿 이하: 1열로 접고, 좌측 안내가 먼저 나온 뒤 폼 패널이 이어진다.
- 모바일: 좌우 여백을 18~24px로 줄이고, 입력 필드는 화면 폭을 안정적으로 채운다.
- 모바일 첫 화면에서 H1, 핵심 본문, 폼 시작부가 겹치거나 잘리지 않아야 한다.
