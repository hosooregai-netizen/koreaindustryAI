## 구성 초안

### Hero

- 핵심 메시지: 현장 데이터와 반복 업무를 실제 업무 방식에 맞는 AI 시스템으로 전환한다.
- CTA: 없음.
- 비주얼: 단일 배경 영상이 아니라 2개 영상 시퀀스로 구성한다.
- 영상 묶음: 1번 영상은 기존 1~3장면을 합친 1묶음으로 약 13초, 2번 영상은 `/assets/hero-landing-reverse.mp4` 기준 약 12초를 사용한다.
- 본문 위치: 데스크톱 홈 히어로 본문은 전체 화면 왼쪽이 아니라 1600px 콘텐츠 기준선의 왼쪽 하단에 배치한다. 1600px 이하에서는 최소 24px 좌우 여백을 유지한다.
- 1묶음 히어로 제목: `현장의 데이터로 만드는 AI 시스템`
- 1묶음 히어로 설명: `회사별 문서 양식, 승인 흐름, 운영 데이터를 반영해 실제 업무 방식에 맞는 시스템을 구성합니다.`
- 2묶음 히어로 제목: `반복 업무를 자동으로 처리하는 AI 시스템`
- 2묶음 히어로 설명: `확인, 분류, 정리, 보고, 결재처럼 매일 반복되는 업무를 AI로 전환합니다.`
- 라벨 정책: 홈 히어로 영상 묶음에는 별도 라벨을 노출하지 않는다.
- 본문 색상: 1묶음은 `#ffffff`, 2묶음은 `#000000`을 사용한다.
- 묶음 전환: 영상이 살짝 축소된 뒤 화면 전체가 왼쪽으로 넘어가는 전환 효과와 텍스트 fade 전환을 사용한다.
- 영상 정책: 모든 영상은 워터마크가 보이지 않도록 크롭 처리된 버전을 사용한다.
- 검증: 모바일 첫 화면에서 제목, 설명, 진행 컨트롤이 겹치지 않는다.

- 동작: 로고 간 간격을 넓게 두고 가로 방향으로 자연스럽게 순환한다.

- 고객사 구성: 한국종합안전(주), (주)볼트앤너트, AnC 기술사 사무소, 금융결제원, 대신자산운용, NH투자증권, 마켓컬리
### 업무 전환 브릿지

- 위치: 신뢰 섹션 다음, 제품 섹션 1 이전에 배치한다.
- 목적: 고객사 로고로 신뢰를 보여준 뒤, 제품명으로 바로 넘어가기 전에 대한산업AI가 해결하려는 업무 문제를 하나의 메시지로 정리한다.
- 역할: Products 섹션의 진입부이다. 좌우 텍스트/이미지 구조가 반복되지 않도록 중앙형 Solution 메시지와 시스템 캔버스로 전환한다.
- 섹션 톤: 흰색 제품 섹션과 자연스럽게 이어지도록 배경을 흰색으로 맞춘다.
- 배경색: `#ffffff`. 보라, 남색, 그라데이션 중심의 장식 배경은 사용하지 않는다.
- 최대 폭: 1600px 콘텐츠 기준선을 유지한다.
- 구성: 상단 중앙 Solution 메시지, 그 아래 와이드 산업 이미지, 하단 AI-Core Flow 시스템 캔버스.
- 반응형: 모바일에서는 모든 요소를 1열로 쌓고, Flow 캔버스 노드도 세로로 전환한다.

#### 텍스트 영역

#### 이미지 영역

- 이미지 방향: 공장, 물류, 사무 운영, 데이터 화면처럼 실제 산업 업무 맥락이 보이는 이미지.

- 처리: 너무 어둡거나 흐린 이미지는 피하고, 실제 업무 환경이 인지될 정도로 선명하게 사용한다.
- 오버레이: 필요하면 아주 약한 어두운 오버레이만 사용한다.

#### AI-Core Flow 캔버스

- 보조 라벨: 없음.
- 제목: `업무의 출발점이 제품 구조로 이어집니다`
- 설명: `흩어진 업무 요소를 하나의 흐름으로 묶고, 필요한 제품 모듈로 자연스럽게 연결합니다.`
- 표시 정책: 데스크톱에서는 제목과 설명이 각각 한 줄로 보이도록 한다. 모바일에서는 자연스럽게 줄바꿈을 허용한다.
- 구성:
  - 입력 노드: `문서` / `양식 수집`, `승인` / `흐름 정리`, `데이터` / `기준화`
  - 중간 노드: `AI-Core` / `업무 구조화` / `데이터 구조화 + 자동화 설계`
  - 도착 노드: `Products` / `Data-Driven / Automation` / `필요한 모듈로 연결`
- 역할: 표처럼 딱딱하게 나뉜 카드 배열이 아니라, 현장의 입력이 AI-Core 구성 단계를 거쳐 Products 섹션으로 이어지는 하나의 흐름처럼 보이게 한다.
- 시각 방향: A안의 유기적인 곡선 파이프라인을 기본으로 하고, C안의 Products 도착 카드 구조를 결합한다.
- 외곽 처리: Flow 전체를 감싸는 큰 카드/박스는 사용하지 않고, 섹션 배경 위에 선과 노드만 떠 있는 형태로 둔다.
- 강조: `AI-Core 업무 구조화` 노드와 `Products Data-Driven / Automation` 노드는 입력 노드보다 크게 보여 제품 진입부 역할을 하게 한다.
- CTA: 없음. 실제 이동은 아래 Products의 제품 이미지와 설명 CTA에서 처리한다.

#### 동작

- 스크롤로 섹션이 보일 때 제목, 설명, 이미지가 순차 fade-up 된다.
- 섹션이 뷰포트 밖으로 벗어났다가 다시 들어오면 fade-up이 다시 재생된다.
- AI-Core Flow 캔버스는 제목 이후 곡선 경로가 왼쪽에서 오른쪽으로 그려지고, 노드들이 좌측부터 순차 fade-up 된다.
- AI-Core Flow도 다시 진입할 때 선 그리기와 노드 fade-up이 다시 재생된다.
- 이미지 hover 시 1.02배 이하로만 미세 확대한다.
- Flow 노드는 hover 시 5px 이하로만 미세하게 떠오른다. 클릭 가능한 요소처럼 보이지 않게 커서와 CTA 스타일은 사용하지 않는다.
- 브릿지 섹션 안에는 버튼, 카드 CTA, 추가 링크를 넣지 않는다.

### 산업 섹션
- 목적: 적용 가능 산업군 별 섹션 제공

- 동작 : 산업별 이미지 hover/focus 시, 카드 이미지가 살짝 확대되고 오른쪽에 검은 그라데이션 오버레이가 들어오며 단색 선형 화살표가 나타난다.

# 3. 페이지별 구성 - Products

## 목적

- AI-Core가 무엇을 하는지 고객 업무 결과 중심으로 설명한다.

## Data-Driven AI-Core 페이지

### Hero

- 목적: 현장 데이터와 문서, 승인 흐름을 기업 맞춤 ERP 구조로 연결한다는 메시지.
- CTA: 문의하기.

### 작동 구조

- 입력: 문서, ERP, 사진, 승인, 운영 데이터.
- 처리: 모듈 조립, 규칙 구성, AI 처리.
- 출력: 업무 화면, 보고서, 대시보드, 알림.

### 적용 예시

- 현장 보고서.
- 견적 검토.
- 문서 정리.
- 운영 데이터 변환.
