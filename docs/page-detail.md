## 구성 초안

### Hero

- 핵심 메시지: 현장 데이터와 반복 업무를 실제 업무 방식에 맞는 AI 시스템으로 전환한다.
- CTA: 없음.
- 비주얼: 단일 배경 영상이 아니라 2개 영상 시퀀스로 구성한다.
- 영상 묶음: 1번 영상은 기존 1~3장면을 합친 1묶음으로 약 13초, 2번 영상은 `/assets/hero-landing-reverse.mp4` 기준 약 12초를 사용한다.
- 본문 위치: 데스크톱 홈 히어로 본문은 전체 화면 왼쪽이 아니라 1600px 콘텐츠 기준선의 왼쪽 하단에 배치한다. 1600px 이하에서는 최소 24px 좌우 여백을 유지한다.
- 1묶음 히어로 제목: `현장의 데이터로 만드는 AI 시스템`
- 1묶음 히어로 설명: `회사별 문서 양식, 승인 흐름, 운영 데이터를 반영해 실제 업무 방식에 맞는 시스템을 구성합니다.`
- 2묶음 히어로 제목: `반복 업무를 자동으로 처리하는 AI 시스템`
- 2묶음 히어로 설명: `확인, 분류, 정리, 보고, 결재처럼 매일 반복되는 업무를 AI로 전환합니다.`
- 라벨 정책: 홈 히어로 영상 묶음에는 별도 라벨을 노출하지 않는다.
- 본문 색상: 1묶음은 `#ffffff`, 2묶음은 `#000000`을 사용한다.
- 묶음 전환: 영상이 살짝 축소된 뒤 화면 전체가 왼쪽으로 넘어가는 전환 효과와 텍스트 fade 전환을 사용한다.
- 영상 정책: 모든 영상은 워터마크가 보이지 않도록 크롭 처리된 버전을 사용한다.
- 검증: 모바일 첫 화면에서 제목, 설명, 진행 컨트롤이 겹치지 않는다.

### 신뢰 

- 목적: 별도 라벨 없이 실제 고객사 및 프로젝트 로고를 순환 노출해 신뢰감을 전달한다.
- 구성: 최대 1600px 폭의 중앙 정렬 로고 마키.
- 동작: 로고 간 간격을 넓게 두고 가로 방향으로 자연스럽게 순환한다.
- 표시 정책: `Project Partners` 같은 섹션명은 노출하지 않는다.
- 고객사 구성: 한국종합안전(주), (주)볼트앤너트, AnC 기술사 사무소, 금융결제원, 대신자산운용, NH투자증권, 마켓컬리
### 업무 전환 브릿지

- 위치: 신뢰 섹션 다음, 제품 섹션 1 이전에 배치한다.
- 목적: 고객사 로고로 신뢰를 보여준 뒤, 제품명으로 바로 넘어가기 전에 대한산업AI가 어떤 방식으로 솔루션을 설계하는지 정리한다.
- 역할: Products 섹션의 진입부이다. 좌우 텍스트/이미지 구조가 반복되지 않도록 상단 중앙 메시지와 하단 3카드 구조로 전환한다.
- 참고 구조: `https://mondrian.ai/kr/ai_infra`의 AI Infra 페이지처럼 상단 중앙 선언문, 충분한 여백, 3개 카드로 이어지는 구성을 참고한다.
- 제외 요소: 참고 페이지의 중간 대형 흐르는 문자 또는 `Overview` 텍스트 헤딩은 사용하지 않는다.
- 섹션 톤: 흰색 제품 섹션과 자연스럽게 이어지도록 배경을 흰색으로 맞춘다.
- 배경색: `#ffffff`. 보라, 남색, 그라데이션 중심의 장식 배경은 사용하지 않는다.
- 최대 폭: 1600px 콘텐츠 기준선을 유지한다.
- 구성: 상단 중앙 Solution 메시지, 큰 호흡의 여백, 하단 3개 다크 카드.
- 반응형: 모바일에서는 텍스트와 카드가 모두 1열로 쌓인다.

#### 텍스트 영역

- 제목: `업무에 맞춘 AI-Core, 제품이 되는 구조까지 설계합니다`
- 설명: `대한산업AI는 회사마다 다른 문서, 승인, 운영 데이터를 먼저 정리하고 반복 처리를 실제 제품 구조로 연결합니다.`
- 표시 정책: 데스크톱에서는 제목이 한 줄에 가깝게 보이도록 넓게 쓰되, 1180px 이하에서는 자연스럽게 줄바꿈을 허용한다.
- CTA: 없음.

#### 카드 영역

- 별도 헤딩: 없음.
- 배치: 상단 메시지 아래 충분한 여백을 두고 3개 카드를 동일한 폭으로 배치한다.
- 카드 톤: 어두운 네이비 카드 위에 푸른 곡선 라인 장식을 넣어 기술적 인상을 만든다.
- 카드 모서리: 사이트 카드 정책에 맞춰 8px radius를 넘기지 않는다.
- 외곽 처리: 3개 카드를 감싸는 별도 큰 카드나 박스는 만들지 않는다.
- 구성:
  - `01` / `업무 구조 정리` / `흩어진 문서 양식, 승인 흐름, 운영 데이터를 실제 처리 단위로 정리합니다.`
  - `02` / `AI-Core 설계` / `데이터 기준, 권한, 화면, 알림, 자동화 조건을 회사 업무 방식에 맞게 조립합니다.`
  - `03` / `제품 모듈 연결` / `정리된 구조를 Data-Driven과 Automation 제품으로 이어 실제 운영 화면으로 확장합니다.`
- CTA: 없음. 실제 이동은 아래 Products의 제품 이미지와 설명 CTA에서 처리한다.

#### 동작

- 스크롤로 섹션이 보일 때 상단 제목과 설명이 먼저 fade-up 된다.
- 섹션이 뷰포트 밖으로 벗어났다가 다시 들어오면 fade-up이 다시 재생된다.
- 3개 카드는 좌측부터 순차 fade-up 된다.
- 카드 hover 시 색상 반전이나 CTA처럼 보이는 변화는 주지 않고, 6px 이하의 미세한 상승과 그림자 변화만 사용한다.
- 브릿지 섹션 안에는 버튼, 카드 CTA, 추가 링크를 넣지 않는다.
