## Data-Driven AI-Core 페이지

### Hero

- 목적: 현장 데이터와 문서, 승인 흐름을 기업 맞춤 ERP 구조로 연결한다는 메시지를 첫 화면에서 명확하게 전달한다.
- 방향: 참고 이미지처럼 밝은 블루 계열 배경 위에 중앙 정렬 히어로를 구성한다. 과한 제품 스크린샷이나 복잡한 장식보다, 제품명이 아니라 고객이 얻는 업무 결과가 먼저 읽히게 한다.
- 배경: 상단은 흰색에 가깝고 중앙으로 갈수록 옅은 블루가 깔리는 부드러운 그라데이션을 사용한다.
- 배경 패턴: 히어로 전체에 아주 옅은 도트 그리드 패턴을 넣는다. 패턴은 텍스트보다 튀지 않게 `opacity`를 낮추고, 모바일에서도 글자 뒤가 지저분해 보이지 않아야 한다.
- 높이: 데스크톱 기준 첫 화면 대부분을 차지하되 다음 섹션의 시작이 살짝 보이게 한다. 모바일에서는 불필요하게 긴 여백 없이 제목과 CTA가 첫 화면 안에 안정적으로 들어온다.
- 정렬: 모든 텍스트와 CTA는 중앙 정렬한다.
- 최대 폭: 텍스트 영역은 약 760px 안에서 정리한다.
- 라벨: `Data-Driven AI-Core`
- 제목: `현장 데이터 걱정 없이 AI 업무 시작`
- 설명: `문서, 승인 흐름, 운영 데이터를 기업 맞춤 ERP 구조로 연결해 업무 화면과 보고서로 바로 활용할 수 있게 합니다.`

  - 설명: `분류, 추출, 정리, 보고서 생성처럼 반복되는 처리 단계를 자동 흐름으로 연결합니다.`
  - 비주얼: `시작 조건`에서 `AI 처리`를 지나 `업무 결과`로 이어지는 가로 플로우 구성.

  - 설명: `자동 처리 결과 중 확인이 필요한 항목만 담당자에게 알리고 승인 또는 수정 이력을 남깁니다.`
  - 비주얼: 알림, 승인, 이력 아이콘이 같은 패널 안에서 정렬되는 구성.

### 자동화 연결 상세

- 위치: 자동화 구성 카드 다음.
- 목적: 단순 기능 나열이 아니라 실제 도입 시 어떤 업무부터 자동화할지 판단할 수 있게 설명한다.
- 구성: Data-Driven 페이지와 같은 2열 교차 섹션처럼 텍스트 블록과 대형 비주얼 블록을 번갈아 배치한다.
- 여백: 각 행 사이에 충분한 세로 여백을 둔다. 행끼리 붙어 보이지 않게 한다.
- 반응형: 모바일에서는 모든 행을 텍스트 먼저, 비주얼 다음 순서로 쌓는다.

#### 상세 1

- 텍스트 위치: 좌측.
- 비주얼 위치: 우측.
- 제목: `작은 반복 업무부터 자동화 흐름을 만듭니다`
- 설명: `처음부터 모든 업무를 자동화하지 않고, 매일 반복되며 기준이 분명한 업무 하나를 먼저 자동 처리 흐름으로 구성합니다.`
- 비주얼: `요청 접수`, `분류`, `정리`, `보고` 단계가 `AI-Core`를 중심으로 순서대로 연결되는 흐름형 일러스트.
- 제외: 검증되지 않은 처리 시간 단축 수치, 인력 절감 수치, 비교 회사명은 사용하지 않는다.
- 포함 문구: `요청 접수`, `분류`, `정리`, `보고`.

#### 상세 2

- 텍스트 위치: 우측.
- 비주얼 위치: 좌측.
- 제목: `예외와 승인은 사람 기준으로 남겨둡니다.`
- 설명: `자동화가 처리할 수 있는 항목과 담당자가 확인해야 하는 항목을 분리해, 승인과 반려 기준이 흐려지지 않게 합니다.`
- 비주얼: 레이어가 쌓인 구조. 위에서부터 `자동 처리`, `예외 조건`, `담당자 확인`, `실행 이력` 순서로 표시한다.
- 표시 정책: 자동화가 모든 판단을 대체한다는 인상을 주지 않는다.

#### 상세 3

- 텍스트 위치: 좌측.
- 비주얼 위치: 우측.
- 제목: `실행 결과와 이력을 한 화면에서 확인합니다`
- 설명: `어떤 업무가 자동 처리됐는지, 어디에서 예외가 발생했는지, 누가 확인했는지를 상태와 이력으로 남깁니다.`
- 비주얼: 자동화 실행 현황 화면처럼 보이는 업무 폼 UI. 상태 배지, 담당자, 실행 시간, 처리 결과, 이력 목록이 보이게 한다.
- 포함 문구: `대기`, `처리 중`, `확인 필요`, `완료`.

- 위치: 자동화 연결 상세 다음, 최종 CTA 이전.
